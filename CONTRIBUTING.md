# Contributing to Finboard

First off, thank you for considering contributing to Finboard! We welcome all contributions, from bug fixes to highly requested features. 

To ensure a smooth and productive collaboration process, please review the following guidelines before you start.

## 📌 Issue Guidelines

### 1. Open an Issue First
Before writing any code, please **open an issue** to discuss your proposed changes or report a bug.
- Check the existing issues to ensure someone hasn't already reported the same problem.
- Provide a clear and valid description. Include steps to reproduce, expected behavior, and screenshots if applicable.

### 2. Respect Issue Assignments
To prevent duplicate work and respect everyone's time, please adhere to the following:
- If you want to work on an issue, comment on it and wait to be assigned.
- **Do not blindly submit a Pull Request** for an issue that is already assigned to someone else. 
- If an assigned issue has seen no activity for a significant period (e.g., over a week), you may ask in the thread if it's okay to take over.

## 🛠️ Development Workflow

Once you are assigned to an issue, follow these steps:

1. **Fork the repository** to your own GitHub account.
2. **Clone the project** locally:
   ```bash
   git clone https://github.com/YOUR_USERNAME/finBoard.git
   ```
3. **Create a new branch** for your feature or bugfix:
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. **Make your changes** and test them thoroughly. Ensure your code aligns with the existing project aesthetic and structure.
5. **Commit your changes** using clear, descriptive commit messages:
   ```bash
   git commit -m "feat: add descriptive feature name"
   ```
6. **Push to your branch**:
   ```bash
   git push origin feature/your-feature-name
   ```
7. **Open a Pull Request** to the main repository. 
   - Please provide a detailed description of your PR and link the associated issue (e.g., `Closes #12`).

## 🤝 Code of Conduct

By participating in this project, you agree to maintain a professional, respectful, and inclusive environment. Treat all contributors with kindness and constructive feedback.

Thank you for helping make Finboard better!
